What it is

Codex

OpenAI's coding agent for software development. It can read projects, write code, run commands, review diffs, debug failures, create artifacts, and help move ideas into shipped prototypes.

Current default

Model guidance

OpenAI's current docs recommend gpt-5.5 for complex coding, tool-heavy agents, grounded assistants, research, and knowledge work. Start with medium reasoning and tune from there.

Prompting Pattern

  • Outcome: what should exist when Codex is done?
  • Context: what files, users, constraints, or examples matter?
  • Guardrails: what should Codex avoid changing or exposing?
  • Verification: what tests, build, screenshots, or artifacts prove success?
  • Handoff: what summary, commit, PR, or next action do you need?

Safety Checklist

  • Keep secrets in environment variables, never in committed code.
  • Use AGENTS.md for durable repo rules and review expectations.
  • Use worktrees and narrow approvals for risky or broad changes.
  • Review diffs, rendered pages, generated artifacts, and sources before shipping.
  • You are accountable for what the agent does on your behalf.
